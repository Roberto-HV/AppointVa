1. Levantar el Sistema en tu Computadora

⚠️
Antes de empezar

Asegúrate de que Docker Desktop esté abierto y corriendo. Puedes verificarlo en la barra de tareas.

Paso 1 — Iniciar PostgreSQL

Abre una terminal PowerShell y ejecuta:

# Si es la primera vez que usas el contenedor:
docker run -d --name postgres-appointva -e POSTGRES_PASSWORD=postgres -p 5433:5432 postgres:16

# Si ya lo creaste antes (uso normal):
docker start postgres-appointva

Paso 2 — Iniciar el Backend

Abre una nueva terminal y ejecuta:

cd c:\Cursos\AppointVa\Back\AppointVaAPI\AppointVaAPI
dotnet run

# Espera hasta ver algo como:
Now listening on: http://localhost:5048
✅
Las migraciones y datos demo se aplican automáticamente

No necesitas correr nada más. El backend crea la base de datos, aplica migraciones y carga los datos de prueba solo.

Paso 3 — Iniciar el Frontend

Abre otra nueva terminal y ejecuta:

cd c:\Cursos\AppointVa\Front
npm run dev

# Verás:
VITE v6.x ready in 500ms
➜ Local: http://localhost:5173/

2. Acceso desde el Celular (misma red WiFi)

📱
¿Para qué sirve esto?

La mayoría de los clientes de un negocio de servicios llegarán desde su celular. Probar en móvil te permite detectar problemas de diseño, botones pequeños, textos difíciles de leer y flujos que no funcionan bien en pantallas chicas.

Paso 1 — Obtener tu IP local

En PowerShell ejecuta:

ipconfig

# Busca la sección de tu adaptador WiFi y anota:
Dirección IPv4. . . . . . . : 192.168.1.XX

Anota tu IP aquí:

Ejemplo: 192.168.1.15

Paso 2 — Iniciar el Frontend en modo red

Detén el frontend si ya está corriendo (Ctrl+C) y vuélvelo a iniciar con --host:

cd c:\Cursos\AppointVa\Front
npm run dev -- --host

# Ahora verás dos URLs:
➜ Local: http://localhost:5173/
➜ Network: http://192.168.1.XX:5173/

Paso 3 — Iniciar el Backend en modo red

Detén el backend (Ctrl+C) y reinícialo con:

cd c:\Cursos\AppointVa\Back\AppointVaAPI\AppointVaAPI
dotnet run --urls "http://0.0.0.0:5048"

Paso 4 — Abrir en el celular

Con el celular en la misma red WiFi, abre el navegador y escribe:

# Booking público (lo que ve el cliente):
http://192.168.1.XX:5173/b/salon-aramis

# Dashboard (propietario):
http://192.168.1.XX:5173/login

Paso 5 — Si el celular no conecta (Firewall)

Ejecuta esto en PowerShell como administrador para abrir los puertos:

netsh advfirewall firewall add rule name="AppointVa Dev" dir=in action=allow protocol=TCP localport=5173,5048
🔧
Solo para desarrollo

Esta regla de firewall es solo para pruebas locales. Cuando termines las pruebas puedes eliminarla con: netsh advfirewall firewall delete rule name="AppointVa Dev"

3. Checklist de Pruebas — Flujo Completo

🔑 Flujo 1 — SuperAdmin

Inicia sesión con admin@appointva.com / Admin123! en /admin

  • Crear un negocio nuevo
    Llena nombre, slug, email. Verifica que aparezca en la lista.
  • Asignar colores personalizados al negocio
    Usa el botón de colores y verifica que se guarden.
  • Activar el negocio
    Sin activarlo el booking público no lo muestra.

🏢 Flujo 2 — Configuración del Negocio (Propietario)

Inicia sesión con aramis@salon-aramis.com / Aramis123!

  • Completar perfil del negocio
    Mi negocio → llenar teléfono, dirección, descripción, zona horaria.
  • Configurar política de cancelación
    Pon 2 horas para probar que funciona. Guardar cambios.
  • Cambiar colores y verificar en booking público
    Cambia el color principal, abre /b/salon-aramis y confirma que se actualizó.
  • Crear un servicio nuevo
    Servicios → Nuevo servicio. Asigna precio, duración y categoría.
  • Crear un empleado nuevo y asignarle servicios
    Empleados → Nuevo empleado. Asigna los servicios que puede realizar.
  • Configurar horario del empleado
    En Mi negocio → Horarios. Activa los días y horas de trabajo.
  • Bloquear un día
    Mi negocio → Días bloqueados. Agrega fecha de mañana y verifica que no aparece disponibilidad en el booking.

📅 Flujo 3 — Booking Público (cliente)

Abre en modo incógnito: http://localhost:5173/b/salon-aramis

  • Completar una reserva de principio a fin
    Servicio → Empleado → Fecha/Hora → Datos personales → Confirmar.
  • Verificar página de confirmación
    Debe mostrar código, botón WhatsApp, Apple Calendar y Google Calendar.
  • Probar el botón de WhatsApp
    Tócalo y verifica que el mensaje prearmado tiene los datos correctos de la cita.
  • Intentar reservar el mismo horario
    Abre otra pestaña incógnita y trata de reservar el mismo slot. Debe rechazarlo.
  • Intentar cancelar dentro de las 2 horas
    Haz una reserva para dentro de 1 hora y trata de cancelarla. El sistema debe bloquearla.
  • Cancelar una cita futura lejana
    Reserva para la próxima semana y cancélala. Debe permitirlo sin problema.
  • Consultar mis citas
    Ir a /b/salon-aramis/mis-citas → ingresar email y teléfono → verificar que aparecen las citas.

📊 Flujo 4 — Dashboard Propietario

  • Ver las citas creadas en CitasPage
    Deben aparecer las citas del cliente con estado Pendiente.
  • Cambiar estado de una cita
    Confirmar → Completar → Inasistencia. Verifica que el cambio se refleja.
  • Crear una cita manual desde el dashboard
    Botón Nueva Cita → llenar datos → confirmar que aparece en la lista.
  • Verificar el cliente en ClientesPage
    El cliente que hizo la reserva pública debe aparecer aquí automáticamente.
  • Revisar Reportes — tab Citas
    Filtrar por fecha y verificar totales (citas, ingresos, completadas, inasistencias).
  • Revisar Reportes — tab Ingresos
    Verificar gráficas por servicio, empleado y día.
  • Exportar CSV y abrirlo en Excel
    Verificar que los acentos y caracteres especiales se ven bien.

👤 Flujo 5 — Control de Acceso por Rol (Empleado)

Inicia sesión con sofia@salon-aramis.com / Sofia123!

  • Verificar que NO puede ver Empleados, Servicios, Clientes, Perfil, Reportes
    Esas opciones no deben aparecer en el sidebar.
  • Intentar entrar directo a /dashboard/empleados
    Escribir la URL manualmente. Debe redirigir automáticamente al dashboard.
  • Verificar que SÍ puede ver Inicio y Citas
    Esas dos secciones sí deben ser accesibles.

🔐 Flujo 6 — Seguridad y 2FA

  • Activar 2FA en /dashboard/seguridad
    Escanear QR con Google Authenticator o similar. Confirmar con el código de 6 dígitos.
  • Cerrar sesión e iniciar de nuevo
    Debe pedir el código TOTP como segundo paso antes de entrar al dashboard.
  • Desactivar 2FA
    Volver a Seguridad → desactivar con código. Confirmar que el siguiente login no pide 2FA.

🏥 Flujo 7 — Health Check

  • Abrir http://localhost:5048/health
    Debe responder: {"estado":"Healthy","checks":[{"nombre":"base-de-datos","estado":"Healthy"}]}

4. Qué Observar en el Celular

📱
Accede desde tu celular a:

http://192.168.1.XX:5173/b/salon-aramis — reemplaza XX con tu IP local

  • Flujo de reserva completo en móvil
    Servicio → Empleado → Fecha → Datos → Confirmar. ¿Es cómodo en pantalla chica?
  • Botones táctiles
    ¿Son lo suficientemente grandes para tocar con el dedo? ¿Hay botones muy juntos?
  • Legibilidad de textos
    ¿Los precios, nombres de servicios y fechas se leen bien?
  • Botón WhatsApp en móvil
    Debe abrir WhatsApp directamente con el mensaje prearmado.
  • Botón Apple Calendar / Google Calendar
    En iPhone debe abrir Calendar. En Android debe abrir Google Calendar.
  • Formulario de datos del cliente
    ¿El teclado no tapa los campos? ¿El input de teléfono muestra teclado numérico?
  • Mis citas en móvil
    Ir a /b/salon-aramis/mis-citas y buscar con email + teléfono. ¿La paginación funciona bien?
  • Dashboard en móvil (propietario)
    Iniciar sesión desde el celular. ¿El sidebar se colapsa? ¿Las tablas son navegables?
